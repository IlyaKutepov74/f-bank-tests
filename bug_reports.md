# Баг-репорты по результатам тестирования F-Bank

## Баг-репорт №1: Перевод с отрицательной суммой принимается как успешный

| Поле | Значение |
|------|----------|
| **Заголовок** | Перевод с отрицательной суммой выполняется без ошибки |
| **Предусловия** | Запущен сервис `http://localhost:8000/?balance=30000&reserved=20001`. Выбран рублёвый счёт. |
| **Шаги воспроизведения** | 1. Кликнуть по карточке «Рубли».<br>2. В поле «Номер карты» ввести 16 цифр (например, `1111 1111 1111 1111`).<br>3. В поле «Сумма перевода» ввести `-500`.<br>4. Нажать кнопку «Перевести». |
| **Ожидаемый результат** | Появляется сообщение об ошибке: «Сумма перевода должна быть положительной». Перевод не выполняется. |
| **Фактический результат** | Появляется alert: «Перевод -500 ₽ на карту 1111 1111 1111 1111 принят банком!» |
| **Приоритет** | Критический |
| **Окружение** | Google Chrome 120, Windows 11, локальный сервер Python 3.11 |

## Баг-репорт №2: Отсутствует валидация номера карты (принимается любая 16-значная последовательность)

| Поле | Значение |
|------|----------|
| **Заголовок** | Некорректный номер карты принимается как валидный |
| **Предусловия** | Сервис запущен, выбран рублёвый счёт. |
| **Шаги воспроизведения** | 1. Кликнуть по карточке «Рубли».<br>2. В поле «Номер карты» ввести `0000 0000 0000 0000` (или любые 16 цифр, не проходящие алгоритм Луна).<br>3. Ввести сумму `100`.<br>4. Нажать «Перевести». |
| **Ожидаемый результат** | Сообщение «Неверный номер карты». Перевод отклонён. |
| **Фактический результат** | Alert: «Перевод 100 ₽ на карту 0000 0000 0000 0000 принят банком!» |
| **Приоритет** | Высокий (нарушение безопасности финансового приложения) |
| **Окружение** | Google Chrome 120, Windows 11 |
